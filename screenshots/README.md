## 截图规范

### 尺寸要求
- **宽度**: 1200px - 2880px
- **高度**: 不限，但建议保持合理比例
- **格式**: PNG 或 JPG
- **大小**: 每张 < 1MB

### 质量要求
- 高清晰度（至少 @2x retina）
- 真实的使用场景
- 清晰的文字
- 适当的标注（如箭头、高亮）

### 内容要求
- 使用虚拟或示例数据
- 不包含真实的 API Token
- 不包含敏感信息
- 界面语言：简体中文

### WordPress 插件目录要求

根据 WordPress 插件目录规范：

1. 截图文件命名为 `screenshot-X.png` (X 为数字)
2. 放在插件根目录的 `assets` 文件夹
3. 在 README.txt 中描述每张截图

## 制作工具推荐

### 截图工具
- **Windows**: Snipping Tool, Snagit
- **Mac**: Screenshot (Command + Shift + 4)
- **跨平台**: ShareX, Lightshot

### 编辑工具
- **简单编辑**: Paint, Preview
- **专业编辑**: Photoshop, GIMP, Figma
- **标注工具**: Skitch, Annotate
